# Storage Analyzer

**Storage Analyzer** είναι Windows desktop εργαλείο σε Python/Tkinter για έλεγχο χώρου ανά προφίλ χρήστη και προαιρετική διαγραφή επιλεγμένων Windows user profiles από διαχειριστή.

Το πρόγραμμα σαρώνει φακέλους χρηστών, υπολογίζει το συνολικό μέγεθος κάθε φακέλου και εντοπίζει τα μεγαλύτερα αρχεία μέσα σε κάθε προφίλ. Μετά τη σάρωση, ο διαχειριστής μπορεί να επιλέξει συγκεκριμένα προφίλ με τικ και να ζητήσει διαγραφή τους.

## Κύριες δυνατότητες

- Σάρωση προεπιλεγμένου φακέλου `C:\Users` ή άλλου φακέλου που επιλέγει ο διαχειριστής.
- Υπολογισμός συνολικού μεγέθους ανά χρήστη.
- Εντοπισμός μεγάλων αρχείων ανά χρήστη.
- Ρύθμιση ελάχιστου μεγέθους αρχείου σε MB.
- Ρύθμιση πλήθους μεγάλων αρχείων που κρατούνται ανά χρήστη.
- Χειροκίνητο export αποτελεσμάτων σε Excel `.xlsx` μόνο όταν πατηθεί το κουμπί **Export Excel**.
- Δεν γίνεται αυτόματη εξαγωγή αρχείων μετά τη σάρωση.
- Δεν υπάρχει PDF export και δεν υπάρχει CSV export.
- Εμφανίζεται μήνυμα ολοκλήρωσης όταν τελειώσει η σάρωση.
- Επιλογή προφίλ για διαγραφή με τικ στη στήλη **Διαγραφή**.
- Κουμπί **Διαγραφή επιλεγμένων** με διπλή επιβεβαίωση και υποχρεωτική πληκτρολόγηση `DELETE`.
- Διαγραφή μέσω `Win32_UserProfile` / PowerShell CIM, όχι απλή διαγραφή φακέλου.
- Καταγραφή αποτελεσμάτων διαγραφής σε `Deletion Log` μέσα στο Excel report.
- Μπλοκάρισμα τρέχοντος/προστατευμένου profile και μπλοκάρισμα profile που τα Windows εμφανίζουν ως `Loaded` ή `Special`.
- Dark UI με custom rounded-style κουμπιά.
- Πιο εμφανή κουμπιά αύξησης/μείωσης για τα αριθμητικά πεδία.
- Υποστήριξη custom εικονιδίου εφαρμογής μέσω `app.ico`.
- Αποφυγή junctions/symlinks/reparse points για περιορισμό διπλομετρήσεων και κύκλων.
- Συνέχιση σάρωσης ακόμα και όταν υπάρχουν `Access Denied` ή άλλα σφάλματα πρόσβασης.

## Ασφάλεια και συμπεριφορά

Η σάρωση είναι μόνο ανάγνωση. Η διαγραφή γίνεται **μόνο** όταν ο διαχειριστής:

1. ολοκληρώσει σάρωση,
2. τικάρει συγκεκριμένα προφίλ,
3. πατήσει **Διαγραφή επιλεγμένων**,
4. επιβεβαιώσει το warning,
5. πληκτρολογήσει ακριβώς `DELETE`.

Το πρόγραμμα δεν κάνει απλή διαγραφή `C:\Users\username`. Για να αποφευχθούν χαλασμένα profile references, προσπαθεί να διαγράψει μέσω Windows `Win32_UserProfile`, ώστε να αφαιρεθεί καθαρά το profile μαζί με τα Windows profile metadata/registry entries.

Σε Windows απαιτείται εκτέλεση ως **Administrator**. Αν το πρόγραμμα ξεκινήσει χωρίς admin δικαιώματα, εμφανίζει μήνυμα και κλείνει.

## Απαιτήσεις source έκδοσης

- Windows 10 ή Windows 11
- Python 3.10+
- `openpyxl` για Excel export
- PowerShell διαθέσιμο στο σύστημα για τη λειτουργία διαγραφής profile

### Αυτόματη εγκατάσταση και εκτέλεση

Για Windows, υπάρχει έτοιμο script που κάνει μόνο του τα βασικά:

```bat
RUN_STORAGE_ANALYZER.bat
```

Το script:

- ζητά αυτόματα Administrator μέσω UAC αν δεν τρέχει ήδη ως διαχειριστής,
- δημιουργεί τοπικό virtual environment `.venv`,
- εγκαθιστά/ενημερώνει τα απαιτούμενα packages από `requirements.txt`,
- ανοίγει απευθείας το Storage Analyzer.

Το `.venv` αγνοείται από το Git μέσω `.gitignore`.

### Χειροκίνητη εγκατάσταση dependencies

```bat
python -m pip install -r requirements.txt
```

## Εκτέλεση από source

Άνοιγμα CMD/PowerShell ως Administrator:

```bat
python storage_analyzer.py
```

## Χρήση

1. Άνοιξε το πρόγραμμα ως Administrator.
2. Άφησε ως φάκελο σάρωσης το `C:\Users` ή επίλεξε άλλο φάκελο.
3. Διάλεξε φάκελο export.
4. Ρύθμισε:
   - πόσα μεγάλα αρχεία να κρατά ανά χρήστη,
   - από πόσα MB και πάνω θεωρείται μεγάλο αρχείο.
5. Πάτησε **Έναρξη σάρωσης**.
6. Όταν τελειώσει η σάρωση, εμφανίζεται μήνυμα ολοκλήρωσης.
7. Για διαγραφή, τικάρεις τα προφίλ στη στήλη **Διαγραφή** και πατάς **Διαγραφή επιλεγμένων**.
8. Για αναφορά, πάτησε **Export Excel**.

## Excel report

Το Excel δημιουργείται με όνομα τύπου:

```text
storage_analyzer_report_YYYYMMDD_HHMMSS.xlsx
```

Περιλαμβάνει:

- `Report Info`
- `User Summary`
- `Large Files`
- `Deletion Log`, μόνο αν έχει γίνει προσπάθεια διαγραφής profile μέσα στο session

## Σημαντικές σημειώσεις

- Η σάρωση μεγάλων προφίλ μπορεί να αργήσει.
- Τα `Access Denied` μετρώνται ως σφάλματα και η σάρωση συνεχίζει.
- Τα junctions/symlinks/reparse points παραλείπονται σκόπιμα.
- Η διαγραφή profile είναι μόνιμη ενέργεια. Πριν χρησιμοποιηθεί παραγωγικά, δοκίμασέ το σε test μηχάνημα ή σε μη κρίσιμο profile.
- Αν ένα profile είναι ενεργό/φορτωμένο (`Loaded=True`), δεν διαγράφεται. Πρέπει πρώτα να γίνει log off ο χρήστης.
